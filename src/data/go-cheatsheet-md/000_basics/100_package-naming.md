## タイトル
title: パッケージの命名規則: コードを整理する名前付け

## タグ
tags: ["basics", "パッケージ", "命名規則"]

## コード
```go
package main // プログラムのエントリーポイントとなる特別なパッケージ

import "fmt"
// import "net/http" // 例: http パッケージ
// import "mypackage/util" // 例: util パッケージ

func main() {
	fmt.Println("This is the main package.")
}

```

## 解説
```text
Goのプログラムは**パッケージ**という単位で構成され、関連コードをまとめます。適切なパッケージ名はコードの可読性・保守性に重要です。

**推奨される命名規則:**
*   短く、簡潔で、内容を表す名前。
*   すべて小文字。
*   可能であれば単一の単語 (例: `fmt`, `os`, `http`)。
*   意味のある名前。
*   アンダースコア (`_`) やハイフン (`-`) は避ける。
*   複数形より単数形を推奨 (例外: `bytes`, `errors`)。
*   `common`, `util` など曖昧な名前は避け、より具体的に (例: `strutil`, `httputil`)。

**パッケージ宣言:**
ソースファイルの先頭で `package` キーワードで所属パッケージを宣言します。コード例は特別な `main` パッケージの宣言です。

**パッケージ名とディレクトリ名:**
通常、パッケージ名はソースコードが置かれている**ディレクトリ名と同じ**にします (例: `net/http` パッケージは `net/http/` ディレクトリ)。これにより `import` 時に場所が明確になります。(`main` パッケージは例外)

分かりやすいパッケージ名はコードの理解を助けます。
