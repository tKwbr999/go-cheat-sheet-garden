## タイトル
title: ブランク識別子 `_`: 使わない値を無視する

## タグ
tags: ["basics", "変数", "ブランク識別子", "_"]

## コード
```go
package main

import (
	"fmt"
	"strconv"
)

func main() {
	// 1. 関数の戻り値を一部無視
	// strconv.Atoi は (int, error) を返す
	numericValue, _ := strconv.Atoi("42") // エラーは無視
	fmt.Println("Value:", numericValue)

	_, err := strconv.Atoi("abc") // 値は無視
	if err != nil {
		fmt.Println("Error:", err)
	}

	// 2. for range でインデックスや値を無視
	nums := []int{10, 20, 30}
	// インデックスを無視
	for _, num := range nums {
		fmt.Println("Num:", num)
	}
	// 値を無視
	for i := range nums { // 値を受け取る変数を省略するとインデックスのみになる
		fmt.Println("Index:", i)
	}
}

```

## 解説
```text
Goでは「宣言したが未使用の変数」はコンパイルエラーになります。
しかし、関数の戻り値など、受け取る必要はあるが使わない値もあります。
そこで使うのが**ブランク識別子 `_`** (アンダースコア) です。

`_` は、代入はされるものの、その値は事実上捨てられ、
未使用変数エラーを回避するための特別な記号です。

**主な使い方:**

1.  **関数の複数の戻り値を一部無視する:**
    関数が複数の値を返す場合、不要な戻り値を `_` で受け取ります。
    コード例では `strconv.Atoi` が返す `error` や `int` を `_` で
    無視しています。
    **注意:** エラーを `_` で無視するのは、エラーが発生しないことが
    確実な場合や、意図的に無視する場合に限定すべきです。

2.  **`for range` ループでインデックスや値を無視する:**
    `for range` でコレクションを反復処理する際、インデックスまたは
    値のどちらか一方だけが必要な場合に、不要な方を `_` で受け取ります。
    コード例では、スライスのループでインデックス (`_`) や値 (`_`) を
    無視しています。(値を無視する場合は `for i := range nums` のように
    値の変数を省略することも可能です)

3.  **`import` での副作用のための利用 (参考):**
    パッケージの関数等を直接使わず、`init` 関数の実行など
    インポートによる副作用のみを目的とする場合、
    `import _ "package/path"` のように書きます。
    (例: `import _ "image/png"`)

ブランク識別子 `_` は、Goの厳格なルールの中で
柔軟なコーディングを可能にする重要な仕組みです。
